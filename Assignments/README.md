# Assignments

## 1. About This Folder

This folder is used to organize the assignments and practice records I completed in the course **CS351/IN321: AI-assisted Software Development**.

The purpose of this folder is not only to store assignment files, but also to record my learning process during the semester.

Through these assignments, I practiced using GitHub, writing documentation, understanding software development workflows, and using AI tools to support coding and debugging.

---

## 2. What I Practiced

During the assignments, I practiced several important skills, including:

* understanding assignment requirements;
* writing and organizing code;
* using GitHub to manage files;
* writing README files and notes;
* testing programs;
* debugging errors;
* using AI tools to support development;
* checking and improving AI-generated results.

These practices helped me understand that software development is not only about finishing the code. It also includes planning, testing, documentation, and reflection.

---

## 3. My Learning Process

At the beginning, I usually focused mainly on whether the program could run successfully.

However, after completing more assignments, I learned that a good assignment should also be easy to understand and easy to explain.

For example, I started to pay more attention to:

* file organization;
* clear explanations;
* meaningful commit messages;
* testing results;
* documentation quality;
* whether the code can be understood later.

This helped me improve my habits when working on software projects.

---

## 4. AI-Assisted Assignment Experience

In this course, I used AI tools to help me complete assignments and understand difficult parts.

AI helped me:

* understand the assignment requirements;
* generate possible solution ideas;
* find bugs in my code;
* explain error messages;
* improve README files;
* organize my learning notes.

However, I also learned that AI-generated answers are not always correct.

Sometimes AI may give code that looks correct but does not fully match the assignment requirements. Sometimes it may ignore edge cases or produce explanations that are too general.

Because of this, I learned that I need to check AI-generated results by myself. I should run the program, test different cases, read the code carefully, and make sure the result is correct.

---

## 5. What I Learned from Assignments

From these assignments, I learned that software development is a step-by-step process.

Each assignment helped me practice a different part of the development process, such as understanding requirements, writing code, testing results, debugging problems, and writing documentation.

The most important thing I learned is that completing an assignment is not only about getting the final answer. I also need to understand how I got the answer and be able to explain my process clearly.

---

## 6. Problems I Met

During the assignments, I encountered some problems.

Some common problems were:

* not fully understanding the requirements at first;
* code not working as expected;
* errors caused by file paths or input format;
* not having enough test cases;
* unclear project structure;
* README files not detailed enough.

These problems helped me understand that careful planning and testing are important.

When I met problems, I tried to solve them by reading error messages, checking the code, testing different inputs, and using AI tools as support.

---

## 7. Improvements I Made

After working on the assignments, I started to improve my workflow.

I tried to:

* organize files more clearly;
* write better explanations;
* test programs before submitting;
* use GitHub commits to record changes;
* check AI-generated code more carefully;
* improve README files and notes.

These improvements helped me make my assignments easier to understand and easier to present.

---

## 8. Reflection

Overall, the assignments helped me build better software development habits.

I learned that writing code is only one part of the process. A good assignment also needs clear requirements, organized files, testing, documentation, and reflection.

I also learned that GitHub and AI tools can help me work more efficiently, but I still need to understand and verify the results by myself.

In the future, I want to continue improving my assignments by writing clearer documentation, adding more test cases, using better commit messages, and organizing my files more carefully.
