## What I Learned

Before taking this course, I usually focused mainly on whether my program could run successfully.

After this course, I learned that software development is not only about writing code. A complete software project also needs clear requirements, readable code, testing, documentation, version control, and reflection.

I also learned that a good repository should be easy for other people to understand. README files, folder names, commit messages, and project structure are all important parts of a software project.

## My GitHub Learning Experience

Through this course, I became more familiar with using GitHub.

I learned how to use GitHub to:

* create repositories
* upload and manage project files
* write README files
* make commits
* use branches
* create pull requests
* merge changes
* check project history

At first, I thought GitHub was mainly for uploading files. Later, I realized that GitHub is also a tool for recording the development process.

By looking at commit history, I can know what I changed at different stages. This helps me review my progress and understand how a project improved step by step.

## My AI-Assisted Development Experience

In this course, I used AI tools to help me with software development.

AI helped me:

* understand project requirements
* generate code ideas
* debug errors
* write explanations
* improve documentation
* organize project structure

However, I also learned that I should not completely rely on AI. Sometimes AI gives code that looks correct, but it may not fully match the project requirements. Sometimes it may also miss edge cases or produce explanations that are too general.

Because of this, I learned that I still need to check the results by myself. I need to read the code, run the program, test different cases, and make sure the final result is correct.

For me, AI is a useful assistant, but I still need to understand the project and make the final decision by myself.

## Projects in This Course

### Project 0: Two Sum

Project 0 was the first project in this course.

The main goal was not only to solve the Two Sum problem, but also to practice the basic software development process. Through this project, I learned how to write code, prepare simple tests, organize files, and use GitHub to record the development process.

This project helped me understand that even a small program can be developed in a structured way.

---

### Project B: CSV Mini Database

Project B was more complete than Project 0.

In this project, I worked with CSV data and designed a small database-like program. I practiced reading data, displaying records, searching data, handling input, and organizing the project structure.

This project helped me understand that when a program becomes larger, structure becomes more important. If the code is not organized well, it becomes harder to debug, test, and explain.
