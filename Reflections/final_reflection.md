# Final Reflection

## 1. What I Learned from This Course

In this course, I learned that software development is not only about writing code.

Before taking this course, I usually focused mainly on whether the program could run successfully. However, after working on the course projects and using GitHub more often, I realized that a complete software project needs more than just working code.

A good software project should also include clear requirements, readable code, testing, documentation, version control, and reflection. These parts help make the project easier to understand, maintain, and improve.

This course helped me understand the importance of the software development process. From project planning to implementation, testing, debugging, and documentation, each step affects the final quality of the project.

---

## 2. My Experience with GitHub

Through this course, I became more familiar with GitHub.

At first, I thought GitHub was mainly a place to upload files. Later, I learned that GitHub is also a tool for recording and managing the development process.

By using commits, I can record what I changed at different stages. This makes it easier to review my progress and understand how the project improved over time.

I also learned that meaningful commit messages are important. If the commit message is clear, I can quickly understand what was changed without checking every file one by one.

In the future, I want to use GitHub more carefully by writing better commit messages, using branches more regularly, and keeping my repositories more organized.

---

## 3. My Experience with AI-Assisted Development

In this course, I used AI tools to help me during software development.

AI helped me understand project requirements, generate code ideas, debug errors, write explanations, and improve documentation. When I did not know how to start a project or how to fix an error, AI helped me think of possible solutions.

However, I also learned that I should not completely rely on AI.

Sometimes AI-generated code looks correct, but it may not fully match the project requirements. It may also miss edge cases or produce explanations that are too general. Because of this, I learned that I still need to check the result by myself.

I need to read the code, run the program, test different cases, and make sure the output is correct. AI can be a useful assistant, but I still need to understand the project and make the final decision by myself.

---

## 4. What I Learned from the Projects

### Project 0: Two Sum

Project 0 helped me practice the basic software development process.

Although the Two Sum problem is not very large, it helped me understand that even a small program should have a clear structure, testing, and documentation.

Through this project, I learned how to explain a problem, design a solution, write code, test the result, and use GitHub to record the development process.

This project also helped me understand that solving the problem is only one part of software development. Explaining the solution and showing the development history are also important.

---

### Project B: CSV Mini Database

Project B helped me understand how to build a more complete small software system.

In this project, I worked with CSV data and practiced reading records, displaying data, searching data, handling input, and organizing the project structure.

Compared with Project 0, Project B required more planning because the program had more functions. I needed to think about how data should be stored, how users interact with the program, and how to handle possible errors.

This project helped me understand that when a program becomes larger, code organization becomes more important. If the code is not organized clearly, it becomes harder to debug, test, and explain.

---

## 5. Mistakes and Improvements

One mistake I made at the beginning was focusing too much on whether the program could run.

I did not pay enough attention to documentation, testing, and repository organization. Later, I realized that these parts are also important because they help other people understand my project.

Another thing I learned is that I should not blindly trust AI-generated results. Even if the answer looks correct, I still need to verify it by testing and reading the code carefully.

Because of these experiences, I started to pay more attention to README files, project structure, test cases, and explanations.

---

## 6. Overall Reflection

Overall, this course helped me understand software development in a more complete way.

I learned that writing code is only one part of a project. A good software project also needs planning, testing, documentation, version control, and continuous improvement.

I also learned how GitHub and AI tools can support the development process. GitHub helps me record and manage my work, while AI helps me solve problems more efficiently. However, both tools need to be used carefully.

In the future, I want to improve my software projects by:

* writing clearer README files;
* using better commit messages;
* organizing files more clearly;
* adding more test cases;
* using branches and pull requests more often;
* checking AI-generated results more carefully.

This course helped me build a better understanding of how to develop, manage, and present a software project.
