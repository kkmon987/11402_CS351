# GitHub Learning Notes

## 1. Why I Learned GitHub

Before this course, I thought GitHub was mainly a place to upload and store files.

After using GitHub in this course, I learned that GitHub is more than file storage. It is a tool for version control, project management, and recording the software development process.

By using GitHub, I can track what I changed, when I changed it, and why I changed it. This helps me understand how my project improves step by step.

---

## 2. What I Learned About Repositories

A repository is a place to store and manage a project.

In this course, I learned that a good repository should be organized clearly. It should include important files such as:

* `README.md`
* source code
* project documents
* test files
* notes
* reflections

I also learned that the folder structure should be easy to understand. If a repository is messy, it becomes harder for other people to read and harder for myself to maintain.

---

## 3. What I Learned About README Files

I learned that `README.md` is one of the most important files in a repository.

A README file should explain:

* what the repository is about
* what the project does
* how the files are organized
* how to run or use the project
* what I learned from the project

Before this course, I did not pay much attention to README files. However, I now understand that a clear README helps other people quickly understand the purpose and structure of a repository.

---

## 4. What I Learned About Commits

Commits are used to record changes in a project.

In this course, I learned that I should not only upload the final version of a project. Instead, I should use commits to record the development process.

A good commit message should clearly explain what was changed. For example:

* `Add initial README`
* `Add project summary`
* `Fix CSV search function`
* `Update final reflection`
* `Organize course notes`

Meaningful commit messages make the project history easier to understand.

---

## 5. What I Learned About Branches

Branches allow me to work on a new feature or change without directly modifying the main version of the project.

For example, I can create a branch to update the README or add a new function. After finishing the changes, I can merge the branch back into the main branch.

Using branches makes the development process safer and more organized.

---

## 6. What I Learned About Pull Requests

A pull request is used to review and merge changes from one branch into another branch.

Through pull requests, I can clearly see:

* what files were changed
* what lines were added or removed
* why the change was made
* whether the change should be merged

Even when working alone, using pull requests can help me practice a more professional software development workflow.

---

## 7. What I Learned About Project History

One important thing I learned is that GitHub can show the history of a project.

By checking the commit history, I can see how the project changed over time. This is useful because it shows the development process, not just the final result.

This also helped me understand that software development is a continuous process. A project is usually improved through many small changes, not completed all at once.

---

## 8. Mistakes I Made

At the beginning, I sometimes focused only on finishing the code and did not pay enough attention to GitHub workflow.

For example, I sometimes forgot to write clear commit messages, or I uploaded files without organizing them carefully.

Later, I realized that GitHub history and repository organization are also important parts of a software project.

---

## 9. How I Want to Improve

In the future, I want to improve my GitHub workflow in several ways:

* write clearer commit messages
* commit changes more regularly
* use branches more often
* create pull requests for important changes
* keep my folders better organized
* avoid unnecessary or duplicated files
* write better README files

These improvements will make my repositories easier to understand and maintain.

---

## 10. Reflection

Through this course, I learned that GitHub is an important tool for software development.

It helps me record my work, manage project versions, organize files, and show my development process.

I also learned that a good project is not only about whether the code works. The repository structure, README file, commit history, and documentation are also important.

In the future, I will try to use GitHub more carefully and professionally in my software projects.
