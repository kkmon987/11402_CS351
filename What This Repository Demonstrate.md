## What This Repository Demonstrates

This repository demonstrates my learning progress in several important ways.

### 1. Understanding Software Development

This repository shows that software development is not only about writing code.

A complete software project should include:

- requirements understanding;
- implementation;
- testing;
- documentation;
- version control;
- reflection.

Through this course, I learned that a good software project needs both working code and a clear development process.

---

### 2. Using GitHub as a Development Tool

This repository also shows my experience using GitHub as a development tool.

GitHub is not only a place to upload files. It is also a tool for recording the development process.

Through commit history, branches, pull requests, and merges, I can track how a project changes over time. This helps me understand what I changed, why I changed it, and how the project improved step by step.

---

### 3. Practicing AI-Assisted Software Development

This repository shows how I used AI tools during the software development process.

AI helped me:

- understand project requirements;
- generate code;
- debug errors;
- write documentation;
- improve project structure.

However, I also learned that AI-generated results must be checked carefully. I cannot only copy the answer from AI. I still need to read the code, test the program, and make sure the result matches the project requirements.

---
