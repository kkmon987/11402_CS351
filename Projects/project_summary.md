# Project Summary

## 1. Overview

This file summarizes the projects I worked on in the course **CS351/IN321: AI-assisted Software Development**.

Through these projects, I learned how to apply software development skills step by step, from a simple algorithm problem to more complete data processing systems.

These projects helped me understand that software development is not only about writing code. It also includes understanding requirements, designing solutions, testing, debugging, writing documentation, using GitHub, and reflecting on the development process.

---

## 2. Project 0: Two Sum

### Project Description

Project 0 was based on the Two Sum problem.

The goal of this project was to find two numbers in a list that add up to a target value.

Although the problem itself was not very large, this project helped me practice the basic software development process.

---

### What I Learned

From Project 0, I learned how to:

* understand a programming problem;
* design a simple algorithm;
* write readable code;
* prepare test cases;
* check edge cases;
* use GitHub to record development history;
* write basic project documentation.

This project helped me understand that even a small program should have a clear structure and testing process.

---

### Reflection

At first, I focused mainly on solving the problem. However, after working on this project, I realized that the development process is also important.

A project should not only produce the correct answer. It should also be easy to understand, easy to test, and easy to explain.

Project 0 helped me build the foundation for later projects.

---

## 3. Project A: JSON Plate-to-Nested Converter

### Project Description

Project A focused on JSON parsing and data transformation.

The goal was to process JSON data and convert it into a nested structure.

---

### What I Learned

From Project A, I learned about:

* JSON parsing;
* data transformation;
* schema assumptions;
* error handling;
* deterministic output;
* golden tests.

This project helped me understand that input format and output correctness are very important in software development.

---

### Reflection

Through this project, I learned that data processing programs need careful design.

If the input data has unexpected format or missing fields, the program should handle the situation properly. I also learned that testing is important because it helps verify whether the output is correct and consistent.

## 4. Project B: CSV Mini Database

### Project Description

Project B focused on building a small CSV-based database and query engine.

In this project, I worked with CSV data and practiced implementing database-like functions.

---

### What I Learned

From Project B, I learned how to:

* read CSV files;
* display records;
* search data;
* design simple query logic;
* think about data structures;
* handle invalid input;
* organize a larger project;
* test different situations.

Compared with Project 0, Project B was more complete because it required more functions and more careful project organization.

---

### Reflection

Project B helped me understand that when a program becomes larger, code organization becomes more important.

If the functions are not separated clearly, the program becomes harder to debug and maintain. This project made me think more about how to organize code, how to handle user input, and how to make the program easier to understand.

I think Project B best shows my progress in this course because it is closer to a small real software system.

## 5. Project C: VCF Variant Filter Engine

### Project Description

Project C focused on structured text parsing and filtering.

The project was related to processing VCF data and applying filter rules.

---

### What I Learned

From Project C, I learned about:

* structured text parsing;
* filtering logic;
* DSL-like filtering;
* correctness testing;

This project helped me understand that different data formats require different parsing and validation methods.

Project C showed me that data processing can become more complex when the format has many rules.
